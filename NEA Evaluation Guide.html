The Final Hurdle

The Evaluation section is where you demonstrate your critical thinking. It's not just about proving your code works; it's about objectively analyzing how it works, what stakeholders think, and where it can go next. Examiners are looking for honesty, not perfection.

1. The Four Pillars

To reach the top band, your evaluation must be comprehensive. It is built upon four distinct pillars, each requiring equal attention and detail. Neglecting one can significantly cap your marks.

  • Post-Development Testing:  Proving robustness.
  • Success Criteria:  The core verdict.
  • Stakeholder Feedback:  User validation.
  • Maintenance:  Future-proofing.

2. Rigorous Testing Workflow

Testing individual modules isn't enough. You must run a full system test against your original plan. Don't hide failures; documenting them shows high-level analytical skills.

3. The Criterion Verdict

This is the core of your evaluation. You must link back to the Analysis section. Honesty is key—examiners prefer a justified "Not Met" over a fake "Met".

Tip: Create a table listing every original criterion. For "Partially Met" or "Not Met" items, explain the technical hurdles or time constraints.

The chart illustrates a realistic, high-scoring project profile. Note that not everything is perfectly "Met".

4. Code Maintainability

Software is never finished. You must prove that another programmer could take over your work. Top band code prioritizes readability and modularity over complex "spaghetti" logic.

Key Attributes of Maintainable Code:

  • 🧩
    Modularity:

    Breaking code into functions/classes (OOP) rather than monolithic blocks.

  • 💬
    Commenting:

    Explaining 'why' logic exists, not just 'what' it does.

  • 🏷️
    Variable Naming:

    Using sensible names (e.g., `calculateTotal`) instead of `x` or `temp`.

5. Stakeholder Feedback

Your project was built for a user. Hand over the finished product and let them drive. Don't guide them.

Analyze their feedback critically. If they found a menu confusing, discuss it in the context of HCI (Human-Computer Interaction) principles.
